# 开源方案速记

## 1. 推荐优先级

### 第一梯队

- `CowAgent`
  适合快速验证“微信聊天 + Agent + 工具调用”

- `weclaw`
  适合做微信到 Agent 的桥接

- `openclaw-weixin`
  适合和 `OpenClaw` 体系配合

### 第二梯队

- `WeChatFerry`
  更适合本地个人微信实验和桌面端能力探索

- `wechaty`
  更底层、更框架化，灵活但要自己补很多东西

## 2. 用法建议

### 如果目标是尽快做出 Demo

优先：

- `CowAgent`
- 自己补一个 `create_draft_order` 工具
- 再接一个确认页

### 如果目标是自己掌控 Agent 编排

优先：

- `weclaw` 或 `openclaw-weixin`
- 自建 Agent API
- 自建工具路由

### 如果目标只是先验证个人微信入口

优先：

- `WeChatFerry`

但这条线不要默认拿去做正式商用。

## 3. 产品边界建议

开源项目可以解决的，主要是：

- 微信消息接入
- Agent 对话
- 工具调用
- 多模型接入

你自己的系统需要补的，主要是：

- 用户偏好模型
- 餐饮/菜单/订单工具
- 确认页和支付页
- 风险控制和日志审计
