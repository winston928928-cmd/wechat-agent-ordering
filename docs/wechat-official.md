# 微信公众号通道

## 当前实现

这一版已经支持最小闭环：

`公众号发文字 -> 我们的服务收消息 -> 调 Agent -> 回复用户`

目前支持：

- 签名校验
- 接口验证
- 订阅欢迎语
- 文本消息转发到 Agent
- 按微信用户绑定独立会话
- 主动回发客服消息

当前不支持：

- 图片、语音、文件消息
- 菜单和卡片
- 多公众号配置

## 两种回复模式

### 1. 被动回复模式

只配置：

- `WECHAT_OFFICIAL_TOKEN`

这时 `POST /wechat/official/callback` 会直接同步返回 XML 文本回复。

适合：

- 本地联调
- 模型响应很快的场景

限制：

- 如果模型太慢，微信侧体验会变差

### 2. 主动回发模式

再额外配置：

- `WECHAT_OFFICIAL_APP_ID`
- `WECHAT_OFFICIAL_APP_SECRET`

这时流程会改成：

1. 微信请求到达回调接口
2. 服务先立即返回 `success`
3. 后台线程继续调模型
4. 用客服消息接口把结果主动发回微信

适合：

- 模型回复时间不稳定
- 想让公众号聊天更稳一点

## 接入步骤

1. 在公众号后台配置服务器地址为你的公网 `https` 地址
2. 路径指向 `/wechat/official/callback`
3. `Token` 填成和 `WECHAT_OFFICIAL_TOKEN` 一样的值
4. 先用文本消息验证收发
5. 如果被动回复偏慢，再补 `AppID / AppSecret` 切到主动回发模式

## 现在的会话策略

- 同一个微信用户会固定绑定到同一个 `session`
- 不同微信用户不会混在一个会话里
- 长期记忆目前还是本地单份自用版，不是多用户独立画像

这一点后面如果要做给别人用，再升级成按用户隔离即可
